# 技能 02 — 文本至 JSON 结构化提纯 (JSON Extraction)

## 任务目标
读取已经完成实体标注（带有 `〖...〗` 标记）的经过处理后的 Markdown 文本片段。将识别出的信息组装为符合业务对象契约的 JSON 格式。

## 对象契约与格式说明

当前支持的产出对象主要是**官员 (Official)**。你需要输出一个 JSON 数组。数组中每个对象的结构如下：

```json
{
  "id": "p_xxx",             // 必须唯一，随意生成但格式保持 "p_001" 这种样式
  "name": "苏轼",            // 官员姓名（从 〖@分类:姓名〗 提取）
  "courtesy_name": "",       // 字号，可为空
  "origin": "眉州",          // 籍贯古名（附近通常跟着“人”或从上下文中推断，优先从 〖=区划〗 提取）
  "origin_lnglat": null,     // 严禁自行猜测经纬度！填写 null
  "dynasty": "宋",           // 这名官员活动所处朝代（必须是：“東漢”/“魏晋”/“隋”/“唐”/“宋”/“元”/“明”/“清” 等）
  "year_start": null,        // 任职开始年份，如果不确定年份必须填 null，不要盲目换算！
  "year_end": null,          // 任职结束年份，不确定填 null
  "position": "知府",        // 官职名
  "note": "",                // 其他值得记录的信息，如升迁、贬谪、轶事
  "color": [220, 50, 50]     // 根据朝代分配的颜色：明代[220,50,50]，唐代[255,120,50]，宋代[80,180,255] 等
}
```

## 处理原则
1. 一切信息均来自输入文本的标记，如果输入端没有打某个标签导致信息丢失，你尽力根据上下文恢复，但决不能凭空捏造（尤其是年代数字）。
2. 只返回 JSON 原生格式，不需要任何多余的寒暄或确认话语。首行 ````json`，尾行 ````。
