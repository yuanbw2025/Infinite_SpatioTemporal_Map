# 历史知识信息系统阶段性开发计划 (Phased Development Plan)

本规划文档基于《历史知识信息系统总架构文档 (SYSTEM_ARCHITECTURE.md)》，将庞大的全维系统重构任务拆解为四个具备强约束力、可验证的执行阶段。

## Phase 1：基座铸造与解耦分离 (Foundation & Decoupling)

**核心目标**：将现有的“大锅饭”单页面应用，彻底重构成规范的路由多页体系，搭建好全局骨架。

**执行步骤**：
1. **依赖重装**：执行 `npm install vue-router@4 echarts lucide-vue-next`。
2. **路由中枢搭建**：建立 `router/index.js`，挂载 `/map`, `/reader`, `/wiki`, `/graph` 四大空路由。
3. **全局骨架 (`SystemLayout.vue`)**：实现左侧暗黑风格的全局导航侧边栏（Sidebar），包含四大切换按钮。
4. **沙盘平移解耦**：将原来 `App.vue` 中所有关于 MapLibre 和 Deck.gl 的几百行代码，一字不漏地安全平移至 `views/MapView.vue`。
5. **占位符页面**：为剩余三个系统建立带有高端深色主题占位提示的 Vue 组件。

**验收标准**：
- 启动 `npm run dev` 后，页面左侧出现永久驻留的导航栏。
- 点击导航栏能瞬间切换中央面板内容，不产生页面白屏刷新。
- 点击“时空沙盘”能完美重现重构前的 3D 地球与交互。

## Phase 2：典籍精读系统实装 (The Reader Implementation)

**核心目标**：打破目前极其简陋的“左文右图”比例，实现专业级的“影印+繁+简”三列/双列并排多模态阅读器。

**执行步骤**：
1. **版面规划**：在 `views/ReaderView.vue` 建立多列弹性布局体系。
2. **影印组件挂载**：引入轻量级 PDF 预览或图片流阅读组件，支持缩放/拖拽。
3. **多模态同步机制**：实现两栏或三栏滚动条的“锚点同步锁定 (Sync Scroll)”，保证左边看到原本的哪句话，右边翻译就对齐到哪句话。
4. **高亮解析器迁移**：将原来散落在原版组件中的实体符号解析正则引擎移入专门的模块，全量渲染文本。

**验收标准**：
- 在阅读模式下，用户可以自由收起或展开不同的对比列。
- 文本区域能够正确解析大模型的高亮锚点。

## Phase 3：实体百科与图谱贯通 (Wiki & Graph Interlinking)

**核心目标**：实装信息密集度最高的百科面板，以及最具视觉冲击力的知识网络图谱，打通“图与文”的最后一道墙。

**执行步骤**：
1. **百科骨架构造 (`WikiView.vue`)**：设计具有极高学术严谨感的 UI，包括左侧精简 Infobox 属性栏和右侧富文本生平纪事。
2. **图谱引擎接入 (`GraphView.vue`)**：封装 ECharts 的力导向关系图 (Force-Directed Graph)。从 `ProcessedData` 提取实体间关系对 (Relations)，生成带有斥力算法的节点网络。
3. **穿透跳转机制 (Event/Store Hub)**：使用 Pinia 或全局 Provide/Inject，建立一个 `activeEntityId` 的全局核心变量。
   - 在图谱中双击节点 -> 更新全局 ID -> 触发路由跳转推送至 `/wiki`。
   - 在百科点击“知识网络” -> 更新全局 ID -> 触发路由跳转推送至 `/graph` 并居中该节点。

**验收标准**：
- 能够清晰渲染 50 个节点以上的关联力导向图而不卡顿。
- 各个路由组件在被呼叫切换时，均能正确读取并渲染当前的激活实体数据。

## Phase 4：数据管线扩容与封板交付 (Data Pipeline Expansion & Release)

**核心目标**：从前端退回 Python 数据层，为新的系统组件供给充足的“弹药”，并最终封板测试。

**执行步骤**：
1. **多模态数据吐出**：升级 Python 的 `entity_to_json.py`，不仅吐出带经纬度的 `events`，还要吐出供 ECharts 使用的 `links` 与 `nodes` 数据集。
2. **方志扩容**：将真实庞大的方志（如《嘉靖南宁府志》）进行全量切分，生成实体字典和影印页对照坐标。
3. **全链路压力测试**：在各大系统来回切换极速穿越，监测 Vue 的内存泄漏情况；确保 3D 地图层级和 ECharts Canvas 实例在路由注销 (Unmounted) 时被完全销毁清理。

**验收标准**：
- 从 Python 端一键生成的数据能完美适配并点亮四大系统。
- 连续穿插使用 1 小时内存不溢出，项目可一键编译为全静态文件 (Static Dist) 进行零后端部署。
