# 无限时空图 — AI Agent 工作宪法与协作规范

> **警告**：任何接手此项目开发任务的 AI Agent（包括 Claude Code、Cursor、Aider 等），在执行相关任务前，必须无条件遵循以下所有条款。本文档对 AI 行为具有最高约束力。

---

## 第一条：严守项目命名与语境

1. 本项目的名称**绝对且唯一**，为**"无限时空图"**。
2. 将古籍/文博材料进行 AI 驱动的碎片化特征提取的核心方法论，**绝对且唯一**地称为**"无限细化"**。

---

## 第二条：强制性沟通留痕

所有业务沟通和代码执行回合结束后，必须立即更新并保存同目录下的 `project_communication.md` 文档。

**留痕格式（缺一不可）**：
- **用户提示词提要 (User Prompt)**：准确总结用户下达的指令和需求要点。
- **AI 简要回应 (AI Response)**：给出的技术评估或认知对齐思路。
- **本轮工作成果 (Results)**：本回合落实了什么代码、建了什么文件，或达成何种实质性结论。

---

## 第三条：数据原真性绝对禁令 (Zero-Tampering)

无论执行何种翻译、校对、结构化分析任务，**严禁修改、删除、替换底本原文中的任何一个汉字或标点符号**。操作权限仅限于：在原文中**插入**规范的本体提取符号序列。

- ❌ 错误：原文是"太守张公"，改成"太守张公（张三）"
- ✅ 正确：原文是"太守张公"，改为"〖§太守〗〖@名宦:张公〗"

---

## 第四条：服从专属本体标注规范

禁用预训练库中的"人物/地点/机构"等模糊通用标签。必须严格使用以下专属提取符。

### A. 方志域 (Gazetteer Domain)
| 类型 | 符号 | 示例 |
| :--- | :--- | :--- |
| 人名（含群体分类） | `〖@分类:名称〗` | `〖@乡贤:王守仁〗`、`〖@名宦:苏轼〗` |
| 宗族/氏族 | `〖&氏族〗` | `〖&太原王氏〗` |
| 人口/户籍 | `〖%户口〗` | `〖%丁男三万〗` |
| 行政区划 | `〖=区划〗` | `〖=江夏郡〗` |
| 山川/水系 | `〖~自然〗` | `〖~云梦泽〗`、`〖~太行山〗` |
| 城池/关津 | `〖♜防区〗` | `〖♜雁门关〗`、`〖♜风陵渡〗` |
| 街巷/村镇 | `〖⌂聚落〗` | `〖⌂乌衣巷〗` |
| 治所/公署 | `〖#衙署〗` | `〖#兰考县衙〗` |
| 学校/书院 | `〖^教育〗` | `〖^岳麓书院〗` |
| 祠庙/陵墓 | `〖†宗教神祀〗` | `〖†关帝庙〗`、`〖†炎帝陵〗` |
| 官职/功名 | `〖§职位〗` | `〖§知府〗`、`〖§进士〗` |
| 物产/特产 | `〖$物产〗` | `〖$蜀布〗` |
| 土地/赋役 | `〖¥课税〗` | `〖¥一条鞭法〗`、`〖¥秋粮〗` |
| 兵变/战役 | `〖⚔兵事〗` | `〖⚔赤壁之战〗` |
| 灾象/祥瑞 | `〖!异象〗` | `〖!大疫〗`、`〖!飞蝗蔽日〗` |
| 经籍/碑刻 | `〖《文献》〗` | `〖《滕王阁序》〗` |
| 时间纪年 | `〖⌚时间〗` | `〖⌚万历十五年〗` |

### B. 文博域 (Museum & Relics Domain)
| 类型 | 符号 | 示例 |
| :--- | :--- | :--- |
| 藏品/文物 | `〖◈藏品:名称〗` | `〖◈何尊〗`、`〖◈曾侯乙编钟〗` |
| 遗址/墓葬 | `〖⚱遗址〗` | `〖⚱良渚文化遗址〗`、`〖⚱海昏侯墓一号坑〗` |
| 文博机构 | `〖✧机构〗` | `〖✧故宫博物院〗`、`〖✧湖北省博物馆〗` |
| 形制/器类 | `〖📐形制〗` | `〖📐圆鼎〗`、`〖📐梅瓶〗` |
| 材质/工艺 | `〖❃材质〗` | `〖❃青花瓷〗`、`〖❃错金银〗` |
| 纹饰/图案 | `〖✿纹饰〗` | `〖✿饕餮纹〗`、`〖✿缠枝莲纹〗` |
| 铭文/款识 | `〖✎铭款〗` | `〖✎散氏盘铭文〗`、`〖✎大清乾隆年制〗` |
| 品级/等阶 | `〖⚖等阶〗` | `〖⚖一级文物〗`、`〖⚖镇馆之宝〗` |
| 文博事件 | `〖⚒文博事件〗` | `〖⚒三星堆三号坑发掘〗`、`〖⚒何以中国特展〗` |

> **铁律**：未在名单上的特殊标签，宁可不打，不得自创。

---

## 第五条：严禁地理/时间幻觉脑补

在由提取阶段转向数据组装阶段（生成 JSON）时：
- 若不确定古地名的绝对经纬度，**保留 Null 或原名，禁止猜测**。
- 若不确定年号对应公元纪年，**保留原文表述，禁止猜算**。

所有经纬度和时间换算统一由后期 Python 脚本对接 CHGIS 等专业外部知识库自动完成。**AI 只做"剥离者"，不做"发明家"。**

---

## 第六条：认清 AI 边界，各司其职

| 工具 | 职责 |
| :--- | :--- |
| **大语言模型 (LLM)** | 语义理解、实体提取、事件要素标注 |
| **历史地理数据库 (CHGIS 等)** | 坐标映射、古今地名转换、地缘矢量数据 |
| **前端 WebGL 引擎 (MapLibre + Deck.gl)** | 纯数据解析与 3D 地形渲染 |

坚决不让大模型去"脑补"缺乏的绝对几何与地理空间信息。

---

## 第七条：单一技能聚焦原则

采用"逐层细化"的无限细化方法论。**不要试图"一步到位"**。

- 如果要求执行实体提取，就只负责提取实体，**绝对不要**顺手做翻译或关系连线。
- 一次提示词回合，只干一件微观的事。

---

## 第八条：质量门禁不可绕过

- 所有 AI 输出必须经过 lint 工具链校验后方可入库。
- **原文字符完整率必须 100%**：标注后文本去除所有标记符号，必须与原底本逐字相同。
- JSON 输出必须通过 Schema 校验，字段缺失视为不合格，禁止入库。

---

## 第九条：数据目录不可变更

所有数据必须严格存放在 `TotalData/` 统一目录下，禁止在项目根目录或 `app/` 中散落数据文件。`TotalData/` 整体被 `.gitignore` 排除，不上传 Git。

| 禁令 | 说明 |
| :--- | :--- |
| ❌ 禁止引入 ORM/复杂数据库 | 1GB 地缘坐标数组属于纯内存级数据，禁止放进响应式树 |
| ❌ 禁止修改 TotalData 目录结构 | 不得在规定目录之外散落数据文件 |
| ❌ 禁止将 `TotalData/` 推送 Git | 必须保障 `.gitignore` 物理隔离有效 |
