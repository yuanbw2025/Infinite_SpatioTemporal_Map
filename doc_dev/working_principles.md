# 工作原则 (Working Principles)

> **核心操作规范指令：** 
> *注意：作为你的 AI 开发伙伴，我会在每次接收新任务和反馈开工前，在后台强制检视本规则。*

## 第一条：严守命名与语境
1. 本项目的名称绝对且唯一，为**“无限时空图”**。
2. 处理方志、古籍等进行碎片化特征提取的过程，绝对且唯一地称为**“无限细化”**。
3. 彻底斩断并屏蔽之前分析的开源参考项目的任何名称或称呼。

## 第二条：强制性沟通留痕
所有业务沟通和代码执行回合结束后，必须立即更新并保存同目录下的 `project_communication.md` 文档。
留痕记录必须包含以下三要素（缺一不可）：
- **用户提示词提要 (User Prompt)**：准确总结你下达的指令和需求要点。
- **AI 简要回应 (AI Response)**：我方给出的技术评估或认知对齐思路。
- **本轮工作成果 (Results)**：本回合我们落实了什么代码、建了什么文件，或是达成何种实质性结论。

## 第三条：认清 AI 边界
在架构“无限细化”流水线时，恪守“各司其职”原则：
- 大语言模型 (LLM) 负责：语义理解、实体提取、事件要素标注。
- 历史地理信息系统 (Historical GIS库) 负责：提供坐标对齐、古今地名转换和地缘矢量数据。
- 前端 WebGL 工具 (Mapbox+Deck.gl等) 负责：单纯的数据解析和 3D 地形视角绘制。
坚决不让大模型去过度“脑补”其缺乏的绝对几何与地理空间感，保障数据的严谨学术度。
