# 无限时空图 — 项目全景规划 (Project Master Plan)

> 用 AI 引擎将古籍方志与文博典藏转化为 3D 交互的结构化知识沙盘。  
> 为历史地理学、博物馆学研究提供基础设施级支撑。

**本文档属性**：项目最高规划准则，含产品愿景、功能规格、技术架构、数据规范、实施阶段和质量标准。**一切技术选型冲突，以本文档为准。**

---

## 一、项目定位与核心能力

本项目是以**中国地方志**与**文博文化遗产**为双重对象的开源知识工程平台。你把一本方志或一份考古报告扔进来，AI 依托九步自动化管线将其拆解为精确到每个人名、地名、年号、事件、藏品元数据的结构化数据，再通过 3D 沙盘地图、时间线、知识图谱等多种方式呈现出来。

### 三大核心能力
1. **AI 知识提纯**：将非结构化的古文、考古报告转化为精确到实体级的结构化知识图谱。
2. **时空精确对齐**：通过历史地理信息系统 (CHGIS) 解决"古今地名偏移"与"朝代纪年换算"，所有坐标有据可查。
3. **沉浸式时空还原**：基于 3D 真实高程地形，还原历史人物足迹、家族迁徙、文物流转的动态全景。

### 开源理念
仓库内包含所有已处理完毕的结构化数据。任何人 `git clone` 后直接打开浏览器即可使用，无需后端、无需 API Key、无需 GPU 服务器。

## 二、产品功能全景

### 2.1 无限时空 3D 沙盘 (The Infinite Spatial Layer)

基于 MapLibre GL JS 与 Deck.gl 打造的 3D 动力学沙盘，在真实高程地形上复原历史博弈。

#### 多图层系统
| 图层 | 数据来源 | 说明 |
| :--- | :--- | :--- |
| **全景层** | 全库所有数据 | 显示已录入数据的热力分布，供全局概览 |
| **人物层** | `persons.json` | 按朝代分级展示历代人物的地理分布密度 |
| **家族层** | `families.json` | 展示望族起源点、迁徙弧线以及世系分支散落 |
| **战争层** | `events.json` | 动态热力光柱标注战场、兵力调动路线及防御关隘 |
| **疆域层** | 历代地理矢量 | 随时间轴自动伸缩的政权疆域与行政区划变迁 |
| **地理变迁层** | `geo_changes/` | 展示河道改道、湖泊消涨、聚落兴废等自然变迁 |
| **文博层** | `sites.json` | 史前遗址、历代墓葬、窑址、馆藏机构的地理分布 |
| **流转层** | `relics.json` | **[核心]** 追踪文物的"生命路径"（出土地 → 辗转流传 → 现馆藏地） |
| **经济层** | `events.json` | 物产分布、商路走向、赋税重心的时空演变 |

#### 核心交互
- **人物足迹追踪**：选中人物，即刻绘制其一生行踪轨迹（发光动态线条）。
- **文物生命追踪**：选中文物，可视化其从发掘点到历史流转点再到博物馆的完整旅程。
- **时间滑块联动**：全局元素随年代平滑切换（地名变更、聚落消长、疆域伸缩）。
- **区域下钻**：点击任意区域，侧边栏推送该区域的历代人物、事件、馆藏详情。
- **人物-文物语义联结**：点击墓葬自动联结方志中的传主信息；点击文物展示方志中的相关记载。

### 2.2 学术级深度标注阅读器 (The Reader Layer)

| 功能 | 说明 |
| :--- | :--- |
| **多类实体语法高亮** | 人名、地名、官职、时间、朝代、制度、物产、赋税、纹饰、铭文等实体按类型着色 |
| **高亮开关** | 可关闭高亮，回归纯净阅读模式 |
| **繁简体切换** | 一键在繁体/简体之间切换 |
| **影印本对照** | 原始影印扫描页与数字化文本左右并排展示 |
| **原文对照** | 标注版与未标注纯文本并排对比 |
| **段落精确引用** | 每个段落有唯一编号（Anchor ID），可复制链接精确定位到任意一句话 |
| **知识索引跳转** | 点击任何实体词条，跳转到该实体的详情页（出处、别名、关联事件） |
| **白话翻译** | 部分章节提供白话对照翻译 |
| **注释层对齐** | 方志中的夹注、考证、按语等按层级展示 |
| **语义引导搜索** | 基于实体属性的深度检索，而非单纯关键字匹配（如：所有含"龙纹"的考古发现） |

### 2.3 实体消歧与知识图谱 (The Graph Backend)

| 模块 | 说明 |
| :--- | :--- |
| **实体总索引** | 所有提取的人名、地名、官职等，带分类、频次、出处 |
| **人名/地名消歧** | 同名异人/异地自动识别和区分（如"信"在不同语境→"韩信"或"蒯通"） |
| **别名合并** | "太守""郡守"等异名同义自动归并 |
| **地名沿革映射** | 建立"地名变迁动态对照库"（如：建康→江宁→应天→南京），结合 CHGIS 精确对应坐标 |
| **事件因果链** | 事件间的因果、延续、包含、对立等9种关系类型 |
| **编年自动换算** | 历代年号、干支纪年与公元纪年 100% 精确转换，处理重名年号问题 |
| **家谱世系网络** | 地方望族、科举家族的谱系演变与姻亲关系图 |
| **层级化元数据标签** | 三级级联标签（如：青铜器→礼器→鼎），优化向量检索精度 |
| **双重视角命名** | 专家定名（如：青花凤穿牡丹纹罐）与公众标签（如：大罐、花瓶）并存 |
| **学术溯源引擎** | 所有结论均保留原文出处锚点，点击即可回溯至阅读器的原始段落 |
| **常识库** | 制度规律、任期规律等，用于反常检测 |

### 2.4 时间线可视化 (The Timeline Layer)

类似"地铁图"的线性可视化：横轴为时间，不同人物/家族/势力为不同线路，事件为站点，交集为换乘站。适合展示人物世系、家族兴衰。

- 多线路并行渲染，无限缩放/拖拽
- 搜索定位：按人名/地名/事件名搜索，自动展开对应线路
- 节点详情面板：点击节点弹出详情，包含原文引用、人物、地点、关联事件

### 2.5 实验与研究工具 (Research Lab)

| 功能 | 说明 |
| :--- | :--- |
| **矛盾分析** | 跨卷/跨库自动检测记载矛盾（如同一人物在不同卷中生卒年不一致） |
| **史源溯源** | 分析方志内容的来源依据链（出自哪部更早的文献） |
| **反常检测** | 基于常识库自动标记不合常理的记载（违反任期规律、官职层级等） |

## 三、技术架构

### 3.1 总体分层原则

```
离线自动化数据工厂 (Python + AI Agent)
          ↕  [生成纯静态 JSON]
纯静态前端应用 (MapLibre + Deck.gl + Vue 3)
          ↕  [浏览器直接加载，零后端]
用户浏览器
```

两层物理隔离，互不耦合。数据加工线在本地离线运行，前端在任意静态托管服务上部署。

### 3.2 前端技术栈
| 组件 | 技术选型 | 职责 |
| :--- | :--- | :--- |
| 3D 地形底图 | **MapLibre GL JS**（完全开源，无 API Key 依赖） | 真实 DEM 高程地形、卫星底图 |
| 数据可视化图层 | **Deck.gl**（MIT 协议） | 轨迹动画、家族弧线、战争热力 |
| UI 框架 | **Vue 3** | 侧边栏、时间轴、图层面板 |
| 在线阅读器 | HTML + CSS + Vanilla JS | 语法高亮、繁简切换、实体跳转 |
| 时间线可视化 | SVG + Vanilla JS | 时间轴线路图 |
| 大规模切片加载 | **PMTiles**（可选） | 海量地理数据按需分片加载 |

> **重要决策**：全系使用 MapLibre GL JS，彻底淘汰需要 API Key 的 Mapbox，确保 100% 零门槛开源。

### 3.3 古地名消歧机制
不同朝代对同一地区的称呼往往完全不同。本项目采用以下三层架构：
1. **AI 只做文字剥离**：大模型在管线中仅标出 `〖=建康〗` 及其上下文朝代信息，不做任何空间推断。
2. **CHGIS 词典底层缝合**：Python 脚本携带 `[地名: 建康, 朝代: 晋]` 查询 CHGIS 数据库，获取唯一 `geo_id` 与经纬度坐标。
3. **前端时空变幻展示**：同一地点在时间滑块下平滑切换地名标签（"建康" → "江宁"），侧边栏展示完整多朝代身世链。

## 五、数据加工线：九步自动化管线

通过 Python 调度 Claude Code，对每份原始材料完成以下九步提纯处理：

```
01 校勘 → 02 结构分析 → 03 实体构建
  → [04 事件 / 05 关系 / 06 本体挂载  并行]
  → 07 逻辑推理与时空挂载 → 08 质量拦截 → 09 前端序列化输出
```

| 步骤 | 职责 |
| :--- | :--- |
| **01 预处理与校勘** | 清理杂字符，繁简统一，字节级完整率校验 |
| **02 结构分析** | 识别卷/章/节结构，为每段赋予全局唯一 Anchor ID |
| **03 本体实体构建** | 依据规范在原文中插入提取符，严禁修改原文字符 |
| **04 事件提取** | 识别人物足迹、行政变迁、发掘发现等核心事件 |
| **05 实体关系构建** | 梳理血缘、师生、藏品权属、政敌等关系网络 |
| **06 本体挂载** | 将实体归入层级化本体树（如文博大类→亚类→细分） |
| **07 逻辑推理与时空挂载** | 对接 CHGIS，将古地名转换为经纬度与公元年 |
| **08 质量拦截 (Lint)** | 校验 JSON 契约完整性，原文字符完整率 100% |
| **09 应用构造** | 产出前端可直接加载的纯静态 JSON 数据集 |

### 技能文档体系 (Skills)
所有处理逻辑固化在 `skills/` 目录下的 Markdown 指令文档中：
- **14个元技能**：通用方法论（反思循环、无限细化、冷启动、质量控制等）
- **~80个管线技能**：具体操作规范（校勘规范、实体标注、年代推断、关系发现等）

## 七、质量保障与 Git 策略

### 7.1 质量门禁矩阵
| 层次 | 标准 |
| :--- | :--- |
| **原文字符完整率** | **100%**：任何 AI 操作后，去掉标注符号所得文本与原底本逐字相同，否则阻止入库 |
| **JSON 契约完整性** | 所有必填字段不得缺失，字段名不得擅改，通过 `jsonschema` 校验 |
| **前端渲染压力** | 10,000 个 TripsLayer 时空节点在中配笔记本上保持 60 FPS |
| **AI 多轮反思** | 提取结果需经 5 轮自动反思审查，每轮修正量递减至收敛 |

### 7.2 分支与部署策略
- `main`：随时可发布，保持纯净
- `feature/skills`：修改大模型指令
- `feature/web`：修改渲染代码
- 部署：Vercel / GitHub Pages（零后端静态构建）

### 7.3 数据分发策略
`TotalData/` 被 `.gitignore` 物理隔离，通过以下渠道分发：
- **Hugging Face Datasets**（学术数据集首选）
- **GitHub Release 附件**（版本绑定发布）
- `README.md` 中附明确下载链接与解压路径说明
