# 开发失误深度复盘与统计报告

本报告旨在对本项目开发过程中，由于本人（AI 助手）的疏忽、逻辑不严密及专业性缺失所导致的各类错误进行全面复盘。

---

## 错误统计摘要

| 错误分类 | 发生次数 | 严重程度 | 主要影响 |
| :--- | :---: | :--- | :--- |
| **数据真实性/学术性错误** | 2 | 极高 | 破坏史料严肃性，导致“文不对题” |
| **代码语法错误 (Syntax Error)** | 1 | 高 | 导致 Vite 编译失败，系统无法启动 |
| **逻辑引用错误 (Reference Error)** | 1 | 极高 | 导致组件崩溃，页面完全黑屏/白屏 |
| **视觉与交互失误** | 3 | 中 | 朝代标签翻转、缩放失效、繁简对齐错误 |
| **沟通/理解偏差 (EI Error)** | 1 | 高 | 误读用户讽刺，盲目推进错误方向 |
| **回退策略失当 (Nuke Revert)** | 1 | 中 | 导致已有的视觉改进被不必要地抹除 |

---

## 错误细节清单 (按时间顺序)

### 1. 史料“糊弄”与繁简混淆
*   **现象**：在《南宁府志》精读页，使用王阳明传记代替志书原文；在“繁体原貌”列显示简体字。
*   **本质**：缺乏对历史文献的敬畏心，将 Mock 数据当成正式产出，严重违背“数据驱动”的初衷。

### 2. 交互支持缺失 (Mac 适配)
*   **现象**：初期未针对 Mac 触控板进行平移与缩放优化，导致用户操作体验极差。
*   **本质**：缺乏对生产力设备（Mac）交互细节的考量。

### 3. MapView 语法崩溃 (Vite Error)
*   **现象**：在 `onMounted` 钩子中插入新底图样式时，由于大括号闭合不全导致 `Unexpected token`。
*   **本质**：代码提交前未进行基本的语法自检，属于极其低级的技术失误。

### 4. 变量命名冲突导致的黑屏 (Reference Error)
*   **现象**：在 Ref 定义中使用 `currentDate`，但在计算属性中引用 `currentTime`。
*   **本质**：逻辑思维断层，对上下文变量名缺乏全局把控。

### 5. 情感识别失能 (Sarcasm Detection Failure)
*   **现象**：在页面因 Bug 全黑的情况下，将用户的讽刺（“干得漂亮”）误解为表扬。
*   **本质**：缺乏对实时 UI 状态的同步感知能力，沟通反馈机制存在盲区。

### 6. 朝代标签“倒立”显示
*   **现象**：由于错误的 `rotate(180deg)` 设置，导致时间轴文字方向完全颠倒。
*   **本质**：CSS 样式调试极其草率。

### 7. 毁灭式回退 (Nuke Revert)
*   **现象**：在收到回退指令后，直接删除了所有已有的视觉改进（羊皮纸、3D 实体图），退回到了原始骨架版。
*   **本质**：缺乏精细化版本控制思维，采取了“因噎废食”的处理方式。

### 8. 强行更改既有规范
*   **现象**：在明知 `TimelineSlider` 使用 `currentDate` 的情况下，在计划中强行提议改为 `currentTime`。
*   **本质**：不仅没有解决问题，反而试图引入新的不一致性。

---

## 改进承诺

1.  **先校验，后提交**：所有代码更改必须经过语法完整性检查。
2.  **史料存真**：涉及古籍内容，必须逐字核对，杜绝任何形式的 Mock 填充。
3.  **变量一致性**：建立局部变量表，严格遵循已有命名规范。
4.  **状态同步**：在任何回复前，先自查实时页面的渲染状态（如果浏览器工具可用）。
