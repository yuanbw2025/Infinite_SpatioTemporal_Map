# 无限时空图：方志与文博高精度知识工程 (Project Whitepaper)

> **用 AI 引擎重新发现历史：将古籍方志与文博典藏转化为 3D 交互的结构化知识沙盘。**
> **为历史地理学、博物馆学研究提供基础设施级的“机床”支撑。**

---

## 1. 项目愿景与核心理念 (Project Vision)

在数字化浪潮中，大量的古籍方志、考古简报、馆藏清册仍处于“数据孤岛”状态。本项目致力于利用 **AI Agent 技术**，实现以下三大突破：
*   **语义提纯**：将非结构化的古文、考古报告转化为精确到实体级的结构化知识。
*   **时空对齐**：通过历史地理信息系统 (HGIS) 解决“古今地名偏移”与“朝代纪年换算”。
*   **沉浸还原**：基于 3D 真实地形，还原历史人物的足迹、家族的迁徙以及文物的流转历程。

---

## 2. 工业级数据生产线：九步自动化管线 (The Pipeline)

本项目构建了严谨的离线数据加工体系，确保每条数据“可审计、不脑补、不篡改”。

### 2.1 数据生命周期流转图
```text
原始书籍/报告 (PDF/TXT/图) -> [1.格式统一] -> [2.AI九步管线处理] -> [3.时空挂载] -> [4.校验入库] -> 应用层
```

### 2.2 九步处理流程详解
1.  **预处理与校勘 (01 Pre-processing)**：清理底本杂字符，进行繁简统一，确保文本字节级完整率 100%。
2.  **结构分析 (02 Structure Analysis)**：自动识别卷、章、节，为每个段落分配全局唯一的**物理锚点 (Anchor ID)**。
3.  **本体实体构建 (03 Entity Extraction)**：在文中插入规范提取符（如 `〖@名宦:苏轼〗`），**严禁修改原文字符**。
4.  **事件提取 (04 Event Extraction)**：识别战争冲突、行政变迁、科举中举、发掘发现等核心事件。
5.  **实体关系构建 (05 Relation Mapping)**：梳理血缘世系、师生门生、藏品权属、政敌盟友等网络。
6.  **本体挂载 (06 Ontology Mounting)**：将提取实体归入层级化本体树（如文博大类->器类->纹饰）。
7.  **逻辑推理与时空挂载 (07 Spatial Mapping)**：对接 CHGIS 等外部库，将古地名自动转换为坐标与公元纪年。
8.  **质量拦截 (08 Quality Matrix)**：基于 Lint 工具链校验 JSON 契约完整性，剔除大模型生成的幻觉内容。
9.  **应用构造 (09 App Serialization)**：产出极致轻量化、支持前端直接加载的纯静态 JSON 数据库。

## 3. 三大核心产品模块 (Product Matrix)

### A. 移动的“无限时空”3D 沙盘 (Infinite Spatial Layer)
基于 MapLibre GL 与 Deck.gl 打造。

| 图层 | 说明 |
| :--- | :--- |
| **全景层** | 全库录入数据的热力分布预览 |
| **人物层** | 按朝代/年份展示某区域的历代人物分布与密集度 |
| **家族层** | 望族起源点、家族成员迁徙弧线以及世系分支图 |
| **战争层** | 动态热力光柱标注战场位置、兵力调动路线及防御要塞 |
| **疆域/图谱层** | 随时间轴自动伸缩的政权范围、历代行政区划变迁（县→府→省） |
| **地理变迁层** | 展示黄河改道、湖泊消涨、古聚落兴废等自然/人文环境变迁 |
| **文博层** | 展示史前遗址、历代墓葬、历代窑址、馆藏机构的地理分布 |
| **流转层** | **[核心交互]** 追踪重量级文物的“生命路径”（出土地 -> 辗转收藏地 -> 现馆藏地） |

### B. 学术级“深度标注”阅读器 (The Reader Layer)
| 功能 | 说明 |
| :--- | :--- |
| **20+ 实体语法高亮** | 人名、地名、官职、物产、年代、纹饰、铭文按类型着色，一键开关 |
| **原文影印对照** | 标准标注版文本与古籍原始影印页左右并排对照，支持同步翻页 |
| **段落精锚点链接** | 每一句话均有唯一链接，支持学术引用时的“深层穿透”回溯 |
| **繁简体/语言切换** | 支持原典繁体与标准化简体的瞬时切换，部分篇目提供白话翻译 |
| **知识穿透跳转** | 点击文中实体词，全自动调取其百科详情、关联事件、出处统计 |
| **语义引导搜索** | 不再是纯文字匹配，而是基于“实体属性”的检索（如：搜索所有含有“龙纹”的考古发现） |

### C. 实体消歧与知识图谱系统 (The Graph Backend)
| 模块 | 说明 |
| :--- | :--- |
| **地名变迁映射** | 建立“地名沿革动态对照库”，实现古今地名的时空归并与消歧 |
| **编年自动转换** | 实现历代年号、干支纪年与公元纪年的 100% 精确自动转换（解决重名年号问题） |
| **家谱/世系模型** | 自动生成地方望族、科考家族的姻亲与血缘拓扑结构 |
| **层级标签体系** | 采用三级级联标签（如：青铜器->礼器->鼎），优化向量检索精度 |
| **双重视角命名** | 兼容专家学术定名（如青花凤穿牡丹纹罐）与公众标签（如场景瓷罐、花瓶） |

## 4. 实验与研究工具 (Advanced Research Lab)

| 工具 | 职责 |
| :--- | :--- |
| **记载矛盾分析** | 自动检测不同文献、不同卷册间对同一人物、事件的记载矛盾 |

## 5. 项目核心优势 (Why Choose Us?)

1.  **全场景通用 (Theme-Agnostic)**：深度兼容“方志域”与“文博域”，只需切换 Schema 即可平移整个工业流水线。
2.  **无限细化逻辑**：坚持“AI 不脑补年代/地图”的铁律，所有关键换算均由代码对接权威 GIS 与历法库。
3.  **零门槛开源 (True Open Source)**：**离线优先架构**。不依赖昂贵的 GPU 服务器或 API Key。全量数据直接入库导出，`git clone` 即可拥有整个数字博物馆。

---

## 6. 快速开始 (Local Setup)

1.  **环境准备**：要求 `Node.js 18+`, `Python 3.10+`。
2.  **获取数据**：按 README 指引下载 `TotalData` 压缩包，解压至项目根目录（该目录已被 `.gitignore` 排除，保证 Git 轻量化）。
3.  **启动开发环境**：在 `app/` 目录下执行 `npm install && npm run dev`。
4.  **数据引擎测试**：安装 `requirements.txt` 后运行 `python scripts/pipeline_runner.py` 开启自动化加工。
