## 一、参考项目数据规模

| 指标 | 数值 |
| :--- | :--- |
| 全文字数 | ~60万字 |
| 章节数 | 130篇（全部标注完毕） |
| 实体数 | 17,000+词条，84,000+次标注 |
| 实体类型 | 23类（19类名词 + 4类动词） |
| 事件数 | 3,000+个，98.7%标注了公元纪年 |
| 事件关系 | 7,600+条，9种类型 |
| 可视化线路 | 130条，3,000+个节点 |
| 技能文档 | 14个元技能 + 78个管线技能 = 共99个 |
| 工具脚本 | 162个（Python为主） |

## 三、核心功能清单（逐项拆解）

### A. 在线阅读器功能

| 功能 | 实现方式 | 技术细节 |
| :--- | :--- | :--- |
| **多类实体语法高亮** | Python渲染器 + CSS | 19类名词（人名/地名/官职/时间/朝代/制度/族群/器物/天文/神话/生物/数量/典籍/思想/身份/邦国/氏族/礼仪/刑法）+ 4类动词（军事/刑罚/政治/经济），每类有独立CSS配色 |
| **高亮开关** | JS设置面板 | 右上角齿轮按钮，可关闭高亮专注阅读，偏好持久化到localStorage |
| **繁简体切换** | 前端JS转换器 | 基于OpenCC规则的即时繁简转换 |
| **拼音标注** | JS + Python预处理 | 古音标注，处理多音字消歧 |
| **知识索引跳转** | 渲染器内建 | 万级实体词条可点击跳转，别名合并，语义消歧 |
| **段落编号系统** | 渲染器 + JS | 每个段落有唯一编号，可精确引用和分享（点击复制链接） |
| **原文对照阅读** | 双文件机制 | 标题旁有"原文"链接，可对照查看未标注的纯净原文 |

| **实体筛选** | 前端JS | 可按实体类型筛选显示 |
| **长对话缩进** | 渲染器后处理 | 超过15字的引号对话自动另起一行、缩进两个汉字宽度 |
| **韵文自动分行** | 渲染器正则检测 | 对规整短句自动在句号后换行显示 |
| **表格渲染** | 渲染器内建 | Markdown表格自动转HTML `<table>`，带行号编号 |

## 四、九步处理管线（核心工作流）

```
01 校勘 → 02 结构分析 → 03 实体构建
  → [04 事件 / 05 关系 / 06 本体 并行]
  → 07 逻辑推理 → 08 质量管理 → 09 应用构造
```

每一步对应一份或多份SKILL技能文档，AI Agent 按照SKILL中的指令严格执行。

## 五、质量保障体系

| 机制 | 说明 |
| :--- | :--- |
| **多轮反思** | AI标注后进行5轮自动反思审查，每轮修正量递减至收敛 |

## 七、对方志项目的复用评估

### 可直接复用 ✅
- **SKILL体系框架**：整套技能文档的组织方式和命名规范
- **AI规则约束模式**：AI工作规范文件的写法
- **九步管线框架**：校勘→结构→实体→事件→关系→本体→推理→管驭→应用
- **14个元技能方法论**：反思、迭代、冷启动、无限细化、质量控制等完全通用
- **lint工具链思路**：文本完整性校验的pattern
- **段落编号系统**

### 需要重写 🔧
- **标注符号体系**：方志需要新增实体类型（如物产、赋税、区划、建筑等）
- **实体类型**：从23类扩展为方志所需的分类体系
- **渲染器**：需要适配新的标注符号和方志特有的排版需求
- **CSS配色**：新增实体类型需要新的颜色方案

### 需要全新开发 🆕
- **3D地图引擎**（Mapbox GL JS + Deck.gl）：参考架构中完全没有GIS功能
- **历史地名坐标词典**：需要外挂历史地理信息系统
- **OCR/多模态处理管线**：处理影印版PDF和图像古籍
- **时间轴联动UI**：拖动时间条看地理变迁
- **人物/家族轨迹追踪功能**
- **多图层系统**：全景/人物/家族/战争/国家/地理变迁等
