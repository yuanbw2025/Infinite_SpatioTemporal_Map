# 方志知识库 — 高精度地方史数据化工程

> 用 AI 将方志史书转化为可交互、可探索、可研究的结构化知识网络。
> 为硕博级别的历史地理学研究提供基础设施级支撑。

---

## 这是什么？

一个以**中国地方志**为对象的开源知识工程项目。

核心目标：你把一本方志扔进来，AI 帮你把它拆成精确到每个人名、地名、年号、事件的结构化数据，然后在浏览器里用 3D 沙盘地图、时间线、家族迁徙图等多种方式呈现出来。

**本项目包含两部分**：
1. **知识库本体**：方志原文的标注、实体提取、事件关系、知识图谱——参照成熟的古籍知识工程方法论构建。
2. **无限时空图**：一个内嵌的 3D 时空可视化工具，在真实地形上展现人物足迹、家族迁徙和地理变迁。

**开源理念**：仓库内包含所有已处理完毕的结构化数据。任何人 `git clone` 后直接打开浏览器即可使用，无需后端、无需 API Key。

## 核心功能

### 一、在线阅读器

参照成熟古籍数字化方案，提供学术级的方志阅读体验：

| 功能 | 说明 |
| :--- | :--- |
| **多类实体语法高亮** | 人名、地名、官职、时间、朝代、制度、物产、赋税、建筑、区划等实体按类型着色，一目了然 |
| **高亮开关** | 可关闭高亮回归纯净阅读模式 |
| **繁简体切换** | 一键在繁体/简体之间切换 |
| **影印本对照** | 原始影印扫描页与数字化文本并列展示 |
| **原文对照** | 标注版与未标注纯文本并排对比 |
| **段落精确引用** | 每个段落有唯一编号，可复制链接精确定位到任意一句话 |
| **知识索引跳转** | 点击任何实体词条，跳转到该实体的详情页（出处、别名、关联事件） |
| **白话翻译** | 部分章节提供白话对照翻译 |
| **注释层对齐** | 方志中的夹注、考证、按语等按层级展示 |
| **语义搜索** | 按人名/地名/官职/事件等维度进行全库搜索 |

### 二、无限时空图（3D 时空可视化）

在浏览器中呈现一个真实地形的 3D 交互地图，如同军事沙盘。

**多图层系统**：

| 图层 | 说明 |
| :--- | :--- |
| **全景层** | 显示所有已录入数据的概览热力 |
| **人物层** | 按朝代分级展示某区域历朝历代的人物分布 |
| **家族层** | 展示家族的起源、迁徙路线、分支散落 |
| **战争层** | 以热力光柱标注主要战场、兵力调动路线 |
| **国家/政权层** | 按时间展示不同政权的疆域范围变化 |
| **行政区划层** | 展示历代行政区划的变迁（县→府→省的演变） |
| **地理变迁层** | 展示河流改道、湖泊消涨、聚落兴废等自然变迁 |
| **经济层** | 物产分布、商路、赋税重心的时空变化 |
| **文化层** | 书院、寺庙、科举中举人数的地理分布 |

**核心交互**：

- **人物足迹追踪**：选中一个人物，地图上即刻绘制出他一生走过的路线，以发光轨迹线动态呈现。
- **家族迁徙追踪**：选中一个家族，可看到从起源地向四方散布的迁徙弧线。
- **时间轴联动**：拖动时间滑块，地图上的聚落随时间"生长"或"消亡"，河流水域随朝代伸缩。
- **区域下钻**：点进某个区域后，侧边栏按朝代分级展示该区域的人物、家族、事件等详细信息。

### 三、知识图谱

| 功能 | 说明 |
| :--- | :--- |
| **实体索引** | 所有提取的人名、地名、官职、时间等，带分类、频次、出处 |
| **人名/地名消歧** | 同名异人/异地自动识别和区分 |
| **别名合并** | "太守""郡守"等异名同义自动归并 |
| **事件链** | 事件之间的因果、延续、包含、对立等9种关系 |
| **编年索引** | 年号→公元年自动转换，编年体数据 |
| **家谱世系** | 地方望族、科举家族的谱系关系 |
| **古今地名对照** | 历史地名→现代地名→经纬度坐标的三层映射 |
| **常识库** | 制度规律、任期规律等，用于反常检测 |

### 四、时间线可视化

类似"地铁图"的线性可视化：横轴为时间，不同人物/家族/势力为不同线路，事件为站点，交集为换乘站。适合展示人物世系、家族兴衰。

### 五、实验与研究工具

| 功能 | 说明 |
| :--- | :--- |
| **矛盾分析** | 跨卷自动检测记载矛盾（如同一人物在不同卷中生卒年不一致） |
| **史源溯源** | 分析方志内容的来源依据链（出自哪部更早的文献） |
| **反常检测** | 基于常识库自动标记不合常理的记载 |

## 技术架构

整个项目分为两大部分：**数据加工线**（离线，由 Claude Code 驱动）和 **前端展示层**（在线，纯浏览器运行）。

### 一、数据加工线（通过 Claude Code 执行，不依赖 API）

项目维护者使用 Claude Code 订阅，在本地直接执行处理。核心方法论是**"无限细化"**——将复杂的古籍处理拆解为极小、极精准的子任务，能用规则和代码约束的地方绝不让 AI 自由生成。

#### 九步管线

```
01 校勘 → 02 结构分析 → 03 实体构建
  → [04 事件 / 05 关系 / 06 本体 并行]
  → 07 逻辑推理 → 08 质量管理 → 09 应用构造
```

#### 全格式投喂处理

```
原始书籍文件（PDF/TXT/EPUB/MOBI/影印图片）
          │
          ▼
┌──────────────────────────────┐
│  步骤一：格式统一             │
│  Python脚本 + 多模态视觉识别  │
│  将任何格式转为标准 Markdown   │
└──────────────┬───────────────┘
               │
               ▼
┌──────────────────────────────┐
│  步骤二：九步管线处理          │
│  Claude Code 按「技能指令」    │
│  逐步执行校勘、标注、提取      │
└──────────────┬───────────────┘
               │
               ▼
┌──────────────────────────────┐
│  步骤三：空间映射              │
│  古地名 → 历史地理信息数据库   │
│  → 经纬度坐标                 │
└──────────────┬───────────────┘
               │
               ▼
┌──────────────────────────────┐
│  步骤四：多轮校验与入库        │
│  时间逻辑校验、坐标合理性检查  │
│  重复实体合并、lint工具链      │
│  → 写入 data/ 目录            │
└──────────────────────────────┘
```

#### 技能文档体系

所有处理逻辑以「技能文档 (SKILL)」的形式固化在 `skills/` 目录下。每个技能是一份 Markdown 指令文档，Claude Code 在执行时读取并严格遵循。

- **14个元技能 (Meta-Skills)**：通用方法论（反思循环、迭代工作流、冷启动、无限细化、质量控制等）
- **~80个管线技能 (Pipeline Skills)**：具体操作指南（校勘规范、实体标注规则、年代推断逻辑、关系发现方法等）

### 二、前端展示层（纯浏览器运行）

| 组件 | 技术选型 | 职责 |
| :--- | :--- | :--- |
| 在线阅读器 | HTML + CSS + Vanilla JS | 语法高亮、繁简切换、实体跳转、段落编号 |
| 3D 地形底图 | **Mapbox GL JS** 或 **MapLibre GL** | 真实 DEM 高程地形、卫星底图 |
| 数据可视化图层 | **Deck.gl** | 人物轨迹、家族弧线、战争热力、聚落多边形 |
| UI 框架 | **Vue 3** 或 **React** | 侧边栏、时间轴、图层面板、搜索框 |
| 时间线可视化 | SVG + Vanilla JS | 时间轴线路图 |
| 大规模数据加载 | **PMTiles**（可选） | 海量地理数据按需分片加载 |

## 工作流

### 对于维护者（你）
1. 将方志原始文件丢进 `raw/` 目录。
2. 打开 Claude Code，按照 `skills/` 中的技能指令执行九步管线处理。
3. 审核 `data/` 和 `kg/` 下生成的结构化数据。
4. `git push`，开源社区即可获得最新数据。

### 对于下载者
1. `git clone` 项目仓库。
2. 在浏览器中打开 `docs/index.html`。
3. 直接使用所有功能：阅读器、3D地图、时间线、知识图谱浏览，无需任何配置。

---

## 核心原则

1. **无限细化**：将复杂的古籍处理拆解为极小、极精准的子任务。能用规则和代码约束的地方，绝不让 AI 自由生成。
2. **大模型不画地图**：大模型负责语义理解和实体提取；空间坐标由历史地理信息数据库提供。二者各司其职。
3. **零门槛开源**：仓库即产品。任何人下载即用，不依赖后端服务和 API Key。
4. **学术级严谨**：面向硕博级别的历史地理学研究，所有数据可溯源、可审计、可修正。
